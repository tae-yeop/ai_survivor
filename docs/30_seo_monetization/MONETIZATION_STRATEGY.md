# Monetization Strategy

Status: Active
Owner: 개인 운영자
Last Updated: 2026-05-09

이 문서는 **AI 시대 생존기 / AI Survivor**의 수익화 원칙과 실행 순서를 정리한다. AdSense만 기다리지 않고, 글의 신뢰도와 독자의 문제 해결을 해치지 않는 범위에서 제휴, 디지털 상품, 문의 리드, 스폰서십을 단계적으로 붙인다.

## 1. 기본 원칙

- 수익화는 본문을 방해하는 광고가 아니라 **실제로 사용했거나 검증한 도구·책·템플릿·장비를 자연스럽게 연결하는 방식**으로 시작한다.
- 아무 제품이나 추천하지 않는다. 튜토리얼, 사용기, 비교 글 안에서 독자가 다음 행동을 할 때 필요한 것만 연결한다.
- AdSense는 장기적으로 깔아두되, 초반 트래픽이 작을 때는 제휴 링크·디지털 상품·컨설팅 리드가 더 빠르게 수익을 만들 수 있다.
- 제휴/스폰서/유료 협업은 본문 상단 또는 링크 근처에 명확히 표시한다.
- 정책과 법령은 바뀔 수 있으므로 Amazon, Coupang, Google, FTC/공정위 가이드는 실제 적용 전 다시 확인한다.

## 2. 추천 우선순위

1. AI 도구 / 개발 도구 제휴 링크
2. 직접 만든 디지털 상품
3. 유료 템플릿 / 스타터킷
4. 컨설팅 / 강의 / 워크숍 리드
5. 스폰서 글 / 스폰서 배너
6. AdSense

## 3. AI·개발 도구 제휴 링크

가장 자연스러운 1순위 수익화다. 블로그 주제가 AI 시대 기술 사용기, 튜토리얼, 바이브코딩이므로 글 속에서 실제로 쓴 도구를 연결하기 쉽다.

적합한 카테고리:

- AI 코딩 도구
- 노트 앱 / 지식관리 툴
- 호스팅 / DB / 스토리지
- 도메인 등록 서비스
- 생산성 SaaS
- PDF/논문 정리 도구
- 자동화 도구

글 연결 예시:

- `Cursor로 블로그 CMS를 만들어본 후기` → 실제로 쓴 에디터, 호스팅, 저장소, 도메인 링크
- `Next.js로 Git-backed 관리자 페이지 만들기` → Vercel, GitHub, 도메인, 관련 강의/책 링크
- `논문 PDF를 AI로 정리하는 워크플로우` → PDF 리더, 노트 앱, AI 도구 링크

운영 규칙:

- 각 서비스가 공식 referral/affiliate 프로그램을 운영하는지 글 작성 시점에 확인한다.
- 프로그램 조건은 자주 바뀌므로 “공식 프로그램이 있으면 연결한다” 정도로 느슨하게 설계한다.
- 추천 이유, 실제 사용 맥락, 장점, 단점, 대안이 없는 링크 박스는 넣지 않는다.

## 4. Coupang Partners / Amazon Associates 제품 링크

제품 링크도 가능하다. 한국어 중심 블로그는 Coupang Partners가 붙기 쉽고, 해외 개발자/영문 콘텐츠까지 확장한다면 Amazon Associates도 후보가 된다.

어울리는 상품군:

- 개발/AI 관련 책
- 키보드, 마우스, 모니터암, 모니터
- 노트북 스탠드, 웹캠, 마이크
- 개발자 책상 세팅
- NAS, 외장 SSD, 백업 장비
- 생산성 관련 태블릿 / 전자책 리더

좋은 배치:

- `AI 코딩할 때 듀얼 모니터가 정말 생산성에 도움이 될까?`
- 실제 세팅, 비용, 장단점, 추천 제품 링크를 함께 제공한다.

나쁜 배치:

- `ChatGPT 프롬프트 작성법` 중간에 무관한 키보드, 영양제, 생활용품 링크를 넣는다.

## 5. 직접 만든 디지털 상품

장기적으로 가장 중요한 수익화 축이다. 이 블로그는 “AI를 직접 써보고 검증한다”는 맥락이 있으므로 글이 쌓일수록 상품화하기 쉽다.

후보:

- AI 도구 비교 체크리스트
- 바이브코딩 프로젝트 문서화 템플릿
- 연구 논문 요약 프롬프트팩
- PyTorch 디버깅 체크리스트
- 10-K / 기업 리서치 자동화 템플릿
- AI 업무 자동화 Notion 템플릿
- 블로그 SEO + AdSense 세팅 체크리스트
- Git-backed 블로그 CMS 스타터킷

예상 상품명:

`AI Survivor Blog Kit`

현재 프로젝트 기준 구성:

- Next.js 블로그 코드
- Git-backed MDX 콘텐츠 저장 구조
- GitHub OAuth 관리자 로그인
- GitHub Contents API 기반 글 저장 흐름
- SEO / RSS / sitemap / ads.txt 설정
- 글 템플릿
- 배포 가이드

주의:

- 현재 활성 구현은 Supabase 기반 CMS가 아니라 GitHub-backed MDX 구조다. Supabase 스키마나 DB CMS는 별도 변형 상품으로 기획할 때만 넣는다.
- 실제 상품화 전에는 라이선스, 재사용 가능한 코드 범위, 비밀값 제거, 설치 문서, 데모 데이터 정리가 필요하다.

## 6. 컨설팅 / 강의 / 워크숍 리드

방문자가 많지 않아도 가능한 수익화다. 결제 기능을 처음부터 만들 필요는 없고, Contact 페이지, Google Form, 이메일 CTA만 있어도 시작할 수 있다.

자연스러운 제안:

- 1:1 AI 도구 세팅 컨설팅
- 개발자용 바이브코딩 워크플로우 세팅
- 연구자용 논문 요약 / 리서치 자동화 세팅
- 직장인용 AI 업무 자동화 워크숍
- 스타트업용 AI 도구 도입 컨설팅

CTA 예시:

> 이 워크플로우를 팀/개인 환경에 맞게 세팅하고 싶다면 문의하세요.

## 7. 스폰서 글 / 스폰서 배너

트래픽이 생긴 뒤 AI 도구, SaaS, 개발 도구 회사와 연결할 수 있는 수익화다.

가능한 형태:

- 주간 뉴스레터 스폰서
- 글 상단 스폰서 박스
- 도구 리뷰 스폰서
- 튜토리얼 공동 제작
- 배너 광고

운영 원칙:

- 스폰서 콘텐츠는 반드시 표시한다.
- 돈을 받더라도 장점만 쓰지 않고 실제 한계와 맞는/맞지 않는 독자를 함께 적는다.
- 광고주가 본문 결론이나 추천 순위를 통제하는 형태는 피한다.

## 8. 제휴·스폰서 표기 규칙

### 8.1 대가성 문구

제휴 링크가 포함된 글은 본문 상단 또는 링크 근처에 이해관계를 표시한다.

기본 문구:

> 이 글에는 제휴 링크가 포함되어 있을 수 있습니다. 링크를 통해 구매하면 작성자가 일정액의 수수료를 받을 수 있으며, 구매자에게 추가 비용은 발생하지 않습니다.

Coupang Partners를 사용할 때는 Coupang이 요구하는 최신 권장 문구를 확인하고 적용한다. 실무 문구 예시는 다음과 같다.

> 이 포스팅은 쿠팡 파트너스 활동의 일환으로, 이에 따른 일정액의 수수료를 제공받습니다.

Amazon Associates를 사용할 때는 Amazon의 필수 식별 문구를 사이트에 명확히 표시한다.

> As an Amazon Associate I earn from qualifying purchases.

### 8.2 링크 속성

제휴 링크, 스폰서 링크, 광고성 링크에는 `rel="sponsored"`를 기본으로 붙인다. 호환성과 보수적 SEO 처리를 위해 `nofollow`도 함께 붙일 수 있다.

```html
<a href="https://example.com" rel="sponsored nofollow" target="_blank">
  제품 보기
</a>
```

### 8.3 얕은 제휴 페이지 금지

다음 패턴은 피한다.

- 제품 설명 복붙
- AI로 만든 일반 추천 글
- 실제 사용 경험 없음
- 장단점 없음
- 가격 비교만 있고 맥락 없음
- 링크가 본문보다 많음
- 제휴 관계를 숨김

권장 구조:

- 왜 이 도구/제품을 썼는지
- 어떤 문제를 해결했는지
- 실제 사용 화면, 사진, 코드, 설정
- 장점
- 단점
- 대안
- 누구에게 추천/비추천인지
- 제휴 링크 명확 표시

## 9. 필요한 기능 후보

바이브코딩으로 수익화 기능을 추가한다면 다음 순서가 좋다.

1. `AffiliateLink` 컴포넌트
2. `ProductCard` 컴포넌트
3. `DisclosureBox` 컴포넌트
4. `SponsorBox` 컴포넌트
5. `/resources` 페이지
6. `/tool-stack` 페이지
7. outbound click tracking

컴포넌트 책임:

- `target="_blank"`
- `rel="sponsored nofollow"`
- 제휴/광고 표기
- 필요한 경우 클릭 이벤트 기록

글 작성 화면에 나중에 추가할 수 있는 필드:

- 이 글에 제휴 링크 포함 여부
- 대가성 문구 자동 표시 여부
- 스폰서 글 여부
- 스폰서명
- 관련 제품 카드
- 관련 도구 카드

## 10. `/resources` 페이지 전략

초기부터 “추천 도구”보다 “Resources” 또는 “내가 쓰는 도구” 톤으로 만든다.

구성:

1. AI 코딩 도구
2. 블로그 개발 스택
3. 리서치 / 논문 정리 도구
4. 업무 자동화 도구
5. 개발자 책상 장비
6. 추천 도서
7. 직접 만든 템플릿

상단 고지:

> 이 페이지에는 제휴 링크가 포함되어 있을 수 있습니다. 실제로 사용했거나 테스트한 도구 위주로 정리합니다.

## 11. 시작 순서

1. AdSense 신청 준비는 계속 진행한다.
2. `/resources` 페이지를 만든다.
3. `AffiliateLink`, `ProductCard`, `DisclosureBox` 컴포넌트를 만든다.
4. Coupang Partners / Amazon Associates / AI 도구 referral 후보를 확인한다.
5. 개발/AI 관련 책, 장비, 도구만 선별해 연결한다.
6. 튜토리얼 글마다 “사용한 도구” 박스를 붙인다.
7. 직접 만든 템플릿/스타터킷은 콘텐츠가 쌓인 뒤 상품화한다.
8. 문의 기반 컨설팅 CTA를 Contact로 연결한다.

권장 본문 흐름:

1. 실제 경험 / 튜토리얼
2. 사용한 도구
3. 관련 글
4. 필요한 경우 제품 / 도구 링크
5. 최종 추천

## 12. 참고 정책

- Google Search Central: paid links에는 `rel="sponsored"` 사용, `nofollow`도 허용.
- Amazon Associates: 사이트에 Amazon Associate 식별 문구를 명확히 표시해야 한다.
- FTC Endorsement Guides: 제휴 수수료처럼 독자의 평가에 영향을 줄 수 있는 관계는 명확하고 눈에 띄게 공개해야 한다.
- 공정거래위원회 추천·보증 심사지침: 경제적 이해관계 공개 원칙과 매체별 표시 방식을 확인해야 한다.
