# Content Strategy

## 1. 콘텐츠 원칙

AI 시대 생존기의 글은 “직접 해본 기록”이어야 한다.

- 도구 이름보다 독자의 문제를 먼저 쓴다.
- 설치, 설정, 에러, 비용, 결과물을 남긴다.
- 성공보다 재현 가능성을 우선한다.
- 추천 대상과 비추천 대상을 분리한다.
- LLM이 만든 일반론은 초안으로만 쓰고, 본문에는 실제 확인한 내용을 남긴다.

## 2. 핵심 필러

### 2-1. AI 도구 사용기

Claude Code, Cursor, ChatGPT, DeepSeek, ElevenLabs 같은 도구를 실제 업무나 제작 흐름에 붙여본다.

핵심 질문: “이 도구를 지금 써도 되는가?”

### 2-2. AI 튜토리얼 검증

인터넷에 떠도는 튜토리얼을 직접 끝까지 따라 한다. 되는 부분, 막히는 부분, 현재 버전에서 달라진 부분을 정리한다.

핵심 질문: “따라 하면 진짜 결과물이 나오는가?”

### 2-3. 바이브코딩 실험실

아이디어에서 서비스 화면, 배포, 수정까지 AI coding workflow를 기록한다.

핵심 질문: “AI와 같이 만들면 어디까지 빨라지고 어디서 깨지는가?”

### 2-4. AI 업무 자동화

반복 업무, 콘텐츠 제작, 문서화, 데이터 처리 같은 흐름을 자동화한다.

핵심 질문: “시간이 실제로 줄었는가?”

### 2-5. 도메인 AI 적용기

주식, 딥러닝, 신약개발 같은 전문 영역은 검증 기준을 더 엄격하게 둔다.

핵심 질문: “도메인 지식 없이 과장하지 않았는가?”

## 3. 글 유형

| 유형 | 목적 | 예시 제목 |
| --- | --- | --- |
| 직접 따라 해봄 | 튜토리얼 검증 | `Claude Code를 다른 provider로 바꿔봤다` |
| 도구 사용기 | 비용/품질/속도 판단 | `ElevenLabs 없이 AI 목소리 만들기` |
| 막힌 부분 정리 | 검색 유입 | `yt-dlp가 특정 사이트에서 실패할 때 확인할 것` |
| 제작기 | 서비스/콘텐츠 산출 | `AI로 블로그 글쓰기 화면 만들기` |
| 업데이트 노트 | 오래된 글 갱신 | `2026년 기준 아직 되는 설정만 다시 확인` |

## 4. 고정 글 구조

```md
# [튜토리얼/도구 이름] 직접 따라 해봤다

## 1. 이 글에서 하려는 것
## 2. 원본 튜토리얼 또는 아이디어
## 3. 준비물
## 4. 실제 진행 과정
## 5. 막힌 부분
## 6. 해결 방법
## 7. 결과물
## 8. 비용과 시간
## 9. 추천/비추천 대상
## 10. 한 줄 결론
```

자세한 작성 템플릿은 `ARTICLE_TEMPLATE.md`를 따른다.

## 5. 첫 글 후보

- Claude Code 비용 때문에 다른 provider로 갈아타기
- DeepSeek 또는 호환 provider를 Claude Code에 연결하기
- 내 목소리 없이 영상 만들기, AI 보이스 생존기
- yt-dlp 다운로드 실패 해결 노트
- AI 글쓰기 에디터를 브라우저에서 직접 만들기
- Vercel에 올린 블로그에서 글쓰기 기능 만들기
- Cursor/Claude Code로 작은 웹앱 끝까지 배포하기
- 이미지/영상 AI 결과물 비용 비교
- AI 도구별 “지금도 되는 설정” 갱신 노트
- 블로그 글 하나를 인스타 카드뉴스/유튜브로 쪼개기

## 6. 품질 체크리스트

- [ ] 제목이 구체적인 문제를 말하는가?
- [ ] 실제 사용 환경과 날짜가 있는가?
- [ ] 설치/설정/프롬프트/코드 중 하나 이상이 있는가?
- [ ] 막힌 부분과 해결 방법이 있는가?
- [ ] 비용, 시간, 결과물 품질이 적혀 있는가?
- [ ] 추천 대상과 비추천 대상이 분리되어 있는가?
- [ ] 다음 행동이 명확한가?
- [ ] 내부 링크가 2개 이상 있는가?
- [ ] 카테고리와 태그가 과하지 않은가?

## 7. 채널 재가공 규칙

블로그 글을 발행한 뒤 다음 단위로 쪼갠다.

- 인스타: 6-7장 카드뉴스
- 유튜브: 실제 따라 하기와 결과물 중심 긴 영상
- 쇼츠/Reels: 막힌 부분 하나와 해결 하나
