## 1. 콘텐츠 원칙

이 블로그의 글은 “AI에 대한 설명”보다 “AI를 써서 실제로 무엇을 해결했는지”를 우선한다.

좋은 글의 기준:

- 문제 상황이 구체적이다.
- 사용한 도구와 환경이 명확하다.
- 과정과 결과가 재현 가능하다.
- 실패와 한계를 숨기지 않는다.
- 독자가 따라 할 수 있는 단계가 있다.
- 제목, URL, 본문이 검색 의도와 일치한다.

---

## 2. 핵심 카테고리

### 2-1. AI 도구 사용기

도구를 소개하는 글이 아니라 실제 업무/개발/학습 문제에 적용한 후기.

예시:

- ChatGPT로 블로그 글 초안을 만들면 실제로 시간이 줄어들까?
- Claude로 긴 문서를 요약할 때 환각을 줄이는 방법
- Perplexity와 ChatGPT로 같은 리서치를 해보고 비교
- NotebookLM으로 논문 묶음을 정리해본 후기

### 2-2. 바이브코딩 실험실

AI 코딩 도구로 실제 기능을 만들고 검증하는 글.

예시:

- Cursor로 정적 블로그를 만들 때 SEO에서 터진 문제
- 바이브코딩으로 sitemap.xml과 RSS 붙이기
- AI가 만든 코드에서 사람이 반드시 확인해야 하는 부분
- Claude Code로 컴포넌트 리팩토링하기

### 2-3. AI 업무 자동화

문서, 리서치, 데이터, 반복 업무를 자동화하는 글.

예시:

- PDF 여러 개를 요약해서 표로 정리하기
- 이메일 초안 생성 워크플로우 만들기
- 주식 리포트 요약 자동화하기
- 논문 요약 프롬프트 템플릿 만들기

### 2-4. AI 시대 개발/생산성 팁

툴 조합, 프롬프트, 검증 루틴, 작업 습관.

예시:

- AI에게 일을 맡기기 전에 문서화해야 하는 것들
- 좋은 프롬프트보다 좋은 작업 단위가 중요한 이유
- AI 코딩 결과를 리뷰하는 체크리스트
- 생산성 도구 조합 추천

### 2-5. 전문 영역 적용기

운영자의 전문성과 연결되는 차별화 카테고리. 초기에는 적게 시작하고, 글 품질이 올라오면 강화한다.

예시:

- PyTorch 에러를 AI로 디버깅하기
- 신약개발 논문을 AI로 요약하고 검증하기
- 미국 주식 10-K를 AI로 분석하기

## 3. 글 유형

| 유형 | 목적 | 예시 제목 |
|---|---|---|
| Tutorial | 독자가 따라 하게 함 | Cursor로 Astro 블로그 만들기 |
| Experiment | 직접 실험 결과 공유 | ChatGPT와 Claude로 같은 코드 만들기 |
| Review | 사용 후 평가 | NotebookLM을 논문 요약에 써본 후기 |
| Troubleshooting | 문제 해결 | sitemap이 Search Console에 안 잡힐 때 |
| Comparison | 선택을 도움 | Cursor vs Claude Code 사용성 비교 |
| Workflow | 반복 가능한 루틴 제공 | 논문 읽기 AI 워크플로우 |

## 4. 발행 우선순위

초기에는 아래 순서로 쓴다.

1. 블로그 자체를 만드는 과정
2. AI 코딩 도구 사용기
3. SEO/AdSense 준비 과정
4. 문서 요약/리서치 자동화
5. 전문 영역 적용기

블로그 제작 과정 자체를 콘텐츠화한다. 즉, “블로그를 만드는 글”이 첫 번째 콘텐츠 시리즈가 된다.

## 5. 첫 10개 글 후보

1. 바이브코딩으로 독립 블로그를 만들 때 먼저 문서화할 것들
2. Cursor로 Astro 블로그를 만들어본 과정과 실패한 점
3. AI 블로그에서 SEO를 망치는 SPA 구조 피하기
4. 블로그 글 frontmatter를 어떻게 설계해야 할까?
5. sitemap.xml, robots.txt, RSS를 한 번에 붙이는 방법
6. ChatGPT로 글 초안을 만들고 사람이 검증하는 루틴
7. Claude로 긴 문서 요약할 때 체크해야 할 환각 패턴
8. Perplexity와 ChatGPT로 AI 도구 리서치를 비교해보기
9. AdSense 신청 전 블로그 체크리스트
10. AI 코딩 결과물을 리뷰하는 나만의 체크리스트

## 6. 글 품질 체크리스트

발행 전 아래를 확인한다.

- [ ] 제목이 구체적인 문제를 말하는가?
- [ ] 본문에 직접 경험이나 실험 결과가 있는가?
- [ ] 사용 환경이 적혀 있는가?
- [ ] 코드, 프롬프트, 스크린샷 중 하나 이상이 있는가?
- [ ] 실패/한계가 적혀 있는가?
- [ ] 독자가 다음 행동을 할 수 있는가?
- [ ] 내부 링크를 2개 이상 넣었는가?
- [ ] 메타 설명이 중복되지 않는가?
- [ ] 카테고리와 태그가 과하지 않은가?
