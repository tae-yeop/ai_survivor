# ADR-004: GitHub-backed Admin Editor

Status: Proposed
Date: 2026-05-06
Owner: 개인 운영자
Depends On: ADR-003 GitHub + MDX Content Workflow

## Context

The current MVP uses Git-backed MDX. This is simple and reliable, but writing posts only through local files or GitHub may become inconvenient. The operator wants the site itself to support login and post writing.

At the same time, the project intentionally moved away from a Supabase database/admin CMS because the blog does not yet need runtime content storage.

## Decision

If browser-based writing becomes necessary, prefer a **GitHub-backed admin editor** instead of a database CMS.

The editor should:

- live behind a private route such as `/admin`
- allow only the owner account
- edit `apps/web/content/posts/<slug>/index.mdx`
- save changes by creating Git commits through GitHub APIs
- keep `status: draft` as the default for new posts
- upload large media to object storage rather than Git
- trigger normal Vercel deployments through Git push/commit

## Preferred Architecture

```text
/admin editor
  ├─ owner login
  ├─ MDX/frontmatter form
  ├─ markdown editor or textarea first
  ├─ media upload panel
  └─ publish button
        ↓
GitHub API commit to repository
        ↓
Vercel Git deployment
```

## Authentication Options

| Option                             | Recommendation       | Notes                                            |
| ---------------------------------- | -------------------- | ------------------------------------------------ |
| GitHub OAuth / Sign in with GitHub | Preferred            | Aligns with Git-backed content and owner account |
| Vercel password protection         | Preview-only helper  | Not enough for production authoring by itself    |
| Supabase Auth                      | Deferred             | Reintroduces the stack ADR-003 avoided           |
| Static secret password             | Avoid for production | Too easy to leak or over-broaden                 |

## Media Storage Options

| Media type       | Initial path                                    | Later path                                              |
| ---------------- | ----------------------------------------------- | ------------------------------------------------------- |
| Optimized images | Git under `apps/web/public/media/posts/<slug>/` | Vercel Blob or Cloudflare R2 if uploads become frequent |
| Long videos      | YouTube/Vimeo/external                          | Vercel Blob/R2 if self-hosting is required              |
| Raw originals    | Local/offline archive                           | Dedicated storage, not Git                              |

## Implementation Phases

### Phase A: No-login workflow, current MVP

- Write in `articles/`.
- Promote to `apps/web/content/posts/<slug>/index.mdx`.
- Commit and push.
- Vercel deploys.

### Phase B: Minimal private editor

- Add `/admin` route but keep it disabled by default.
- Add owner-only login.
- List files from GitHub.
- Edit frontmatter/body in a simple form.
- Save as Git commit with `status: draft`.

### Phase C: Media upload

- Add Vercel Blob or another object store.
- Store final URLs in MDX frontmatter/body.
- Keep large files out of Git.

### Phase D: Publish controls

- Add preview, publish, archive, scheduled-date checks.
- Keep published output generated by the normal build pipeline.

## Non-Negotiables

- The admin route must not expose write APIs without owner authentication.
- New posts default to `draft`.
- A failed commit or upload must show a visible error; do not silently drop content.
- The public site must keep working without the admin editor.
- The editor must not require Supabase unless a new ADR explicitly chooses it.

## Open Questions

- Which login provider should be used: GitHub OAuth, NextAuth/Auth.js, Clerk, or Vercel Marketplace auth?
- Should admin commits go directly to `master` or to a draft branch/PR?
- Which media store is best after the first upload-heavy month: Vercel Blob, Cloudflare R2, or YouTube/Vimeo for video?
- Should the editor use plain Markdown first, or a richer editor later?

## References

- GitHub REST API can create or update repository file contents by committing Base64 content to a path.
- Vercel Blob supports images, screenshots, cover images, videos, and large files for display/download.
