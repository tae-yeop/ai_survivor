# ADR-001: Static Content First Blog

Status: Accepted  
Date: 2026-05-05

## Context

이 블로그의 핵심 목적은 AI 도구, 바이브코딩, 업무 자동화, 기술 실험 글을 검색엔진과 독자가 쉽게 읽을 수 있게 제공하는 것이다.

초기에는 로그인, 댓글, CMS, DB, 실시간 기능이 필요하지 않다.

## Decision

MVP는 정적 콘텐츠 우선 구조로 만든다.

- 글은 Markdown/MDX로 관리한다.
- 글 상세와 목록은 정적 HTML로 생성한다.
- SEO 메타데이터, sitemap, RSS는 빌드 시 생성한다.
- 광고는 승인 후 활성화하며, 초기에는 광고 슬롯 컴포넌트만 준비한다.

## Consequences

장점:

- 빠르다.
- 구현이 단순하다.
- SEO에 유리한 정적 HTML을 만들기 쉽다.
- 운영자가 글을 Git 기반으로 관리하기 쉽다.
- 호스팅 비용이 낮다.

단점:

- 비개발자 CMS 경험은 부족하다.
- 댓글/회원 기능을 추가하려면 별도 서비스나 DB가 필요하다.
- 글 작성 워크플로우가 코드 저장소 중심이다.

## Revisit Trigger

아래 조건 중 하나가 생기면 재검토한다.

- 댓글/회원/저장 기능이 필요해진다.
- 운영자가 Git 기반 글 작성이 불편해진다.
- 여러 명이 글을 쓰게 된다.
- 자체 검색이나 추천 기능이 중요해진다.
