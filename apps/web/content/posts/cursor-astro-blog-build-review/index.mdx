<h2>결론 먼저</h2>
<p>
  Cursor에 문서를 넘겨주니 페이지와 라우팅은 빠르게 나왔지만, 글 본문을 클라이언트에서 fetch해
  보여주는 SPA 패턴을 제안하는 순간이 있었다.
</p>

<h2>SEO에서 막힌 부분</h2>
<p>
  글 상세는 검색엔진이 읽을 수 있는 서버 HTML이 우선이다. 검색 유입을 목표로 하는 블로그라면 본문은
  JavaScript가 꺼져도 읽혀야 한다.
</p>

<h2>배운 점</h2>
<ul>
  <li>샘플 데이터는 Phase 1까지만 유효하다.</li>
  <li>본문 저장소와 렌더링 책임을 분리해야 한다.</li>
  <li>빌드 단계에서 draft와 scheduled 노출을 검증해야 한다.</li>
</ul>

<h2>Next.js 전환 기준</h2>
<p>
  Next.js App Router에서는 파일 기반 content loader를 서버 컴포넌트에서 직접 호출하고, 공개 slug만
  정적 파라미터로 생성하는 방식이 가장 단순했다.
</p>
