안녕하세요. AI 생존러입니다.

이번 글은 **Claude Code의 토큰 비용을 줄이는 실험**에 대한 메모입니다. Claude Code는 정말 편한 도구지만, 자동화 루프나 대규모 리팩터링을 돌리다 보면 토큰이 훅 빠지는 게 체감되거든요. 그래서 Anthropic 호환 엔드포인트를 제공하는 **DeepSeek**으로 라우팅해서 작업 일부를 돌리고 있습니다.

처음 세팅할 땐 "이거 안 망가지나?" 싶었는데, 환경변수 몇 개만 잡아주면 끝납니다. 복붙하고 바로 실행할 수 있게 정리했습니다.

## [0] 이 글 핵심 요약

| 항목 | 내용 |
| --- | --- |
| 무엇을 한다 | Claude Code를 Anthropic 대신 DeepSeek로 라우팅 |
| 어떻게 | `ANTHROPIC_BASE_URL` + `ANTHROPIC_AUTH_TOKEN` + 모델 환경변수 교체 |

## [1] 왜 DeepSeek로 라우팅?

### 1-1. 비용 문제

Claude Code는 작업 품질이 좋습니다. 그런데 이런 작업을 돌리기 시작하면 토큰이 빠르게 소모됩니다.

- 오래 실행되는 자동화 루프
- 대규모 리팩터링
- 실패 → 수정 → 재시도가 반복되는 루프

환율까지 끼면 개인 프로젝트엔 부담이 적지 않습니다.

### 1-2. 어떤 작업에 적합한가

작업을 두 종류로 나눠두면 마음이 편합니다.

**대체 프로바이더로 돌려도 되는 작업**
- 아이디어 정리, 초안 작성
- 코드 탐색, 단순 리팩터링
- 비용 한도 걸어두고 여러 번 실험하는 케이스
- 모델 자체를 비교해보고 싶을 때

**기본 환경 그대로 쓸 작업**
- 배포 직전 보안 수정
- 결제/권한 로직
- 손실 큰 마이그레이션
- 한 번 잘못 돌리면 복구가 어려운 작업

> 핵심 정리. **"실패해도 다시 돌릴 수 있는 작업"에만 대체 프로바이더를 켭니다.**

---

## [2] 사전 준비

DeepSeek 플랫폼에서 API 키를 발급받습니다. 그리고 **반드시 일일 예산을 같이 걸어두세요.** 코딩 에이전트는 자동 재시도와 긴 루프 때문에 예상보다 토큰이 훨씬 빨리 빠집니다. 새벽에 자동화 하나가 미친 듯이 재시도하다가 잔액을 다 태우는 일이 실제로 일어납니다.

API 키는 절대 레포에 커밋하지 마세요. `.env`, 스크립트, 블로그 예제에는 `<YOUR_DEEPSEEK_API_KEY>` 같은 placeholder만 남겨둡니다.

> 핵심 정리. **API 키 발급 → 일일 예산 설정 → 키는 환경변수/시크릿 매니저로만 관리.**

## [3] Windows PowerShell 세팅

### 3-1. 일회성 테스트 (복붙하고 바로 실행)

처음엔 이 방식을 추천합니다. 새 PowerShell 창을 띄우면 자동으로 사라지니까 안전합니다.

```powershell
$env:ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
$env:ANTHROPIC_AUTH_TOKEN="<YOUR_DEEPSEEK_API_KEY>"

$env:ANTHROPIC_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_OPUS_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL="deepseek-v4-pro[1m]"
$env:ANTHROPIC_DEFAULT_HAIKU_MODEL="deepseek-v4-pro[1m]"
$env:CLAUDE_CODE_SUBAGENT_MODEL="deepseek-v4-pro[1m]"

$env:ENABLE_TOOL_SEARCH="false"

cd C:\path\to\your-project
claude
```

`ANTHROPIC_DEFAULT_*_MODEL`을 **전부 같은 모델명으로 맞춰주는 게 포인트**입니다. 안 그러면 Opus 호출은 DeepSeek로 가고 서브에이전트는 다른 곳을 찾는 이상한 상태가 됩니다.

### 3-2. PowerShell 프로필에 함수로 저장

매번 입력하기 귀찮다면 PowerShell 프로필에 함수로 박아둡니다. 단, **default `claude`는 건드리지 말고 별도 함수로** 만들어 두는 걸 추천합니다.

먼저 프로필 파일 생성.

```powershell
New-Item -ItemType File -Path $PROFILE -Force
notepad $PROFILE
```

아래 함수를 추가합니다.

```powershell
function Use-DeepSeekClaude {
  $env:ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
  $env:ANTHROPIC_AUTH_TOKEN="<YOUR_DEEPSEEK_API_KEY>"
  $env:ANTHROPIC_MODEL="deepseek-v4-pro[1m]"
  $env:ANTHROPIC_DEFAULT_OPUS_MODEL="deepseek-v4-pro[1m]"
  $env:ANTHROPIC_DEFAULT_SONNET_MODEL="deepseek-v4-pro[1m]"
  $env:ANTHROPIC_DEFAULT_HAIKU_MODEL="deepseek-v4-pro[1m]"
  $env:CLAUDE_CODE_SUBAGENT_MODEL="deepseek-v4-pro[1m]"
  $env:ENABLE_TOOL_SEARCH="false"
  claude
}
```

이러면 평소엔 `claude`로 Anthropic, 필요할 때만 `Use-DeepSeekClaude`로 DeepSeek로 전환할 수 있습니다.

> Tip. API 키를 프로필에 평문으로 박아두는 게 찜찜하다면 Windows Credential Manager나 별도 secret store에서 읽어오게 바꾸는 걸 권장드립니다.

> 핵심 정리. **default 분리 + 함수 1개로 토글.**

## [4] macOS / Linux 세팅

셸에서 일회성 테스트는 아래처럼.

```bash
export ANTHROPIC_BASE_URL="https://api.deepseek.com/anthropic"
export ANTHROPIC_AUTH_TOKEN="<YOUR_DEEPSEEK_API_KEY>"

export ANTHROPIC_MODEL="deepseek-v4-pro[1m]"
export ANTHROPIC_DEFAULT_OPUS_MODEL="deepseek-v4-pro[1m]"
export ANTHROPIC_DEFAULT_SONNET_MODEL="deepseek-v4-pro[1m]"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="deepseek-v4-pro[1m]"
export CLAUDE_CODE_SUBAGENT_MODEL="deepseek-v4-pro[1m]"

export ENABLE_TOOL_SEARCH=false

cd /path/to/your-project
claude
```

영구로 쓰려면 `~/.zshrc`나 `~/.bashrc`에 위 PowerShell 예시처럼 **함수로 감싸서** 넣습니다. 기본 `claude` 사용과 안 섞이도록.

## [5] 정상 연결 확인 + 트러블슈팅

Claude Code가 열리면 `/status`로 현재 모델과 연결 상태를 확인합니다. 모델명이 `deepseek-v4-pro[1m]`로 잡혀 있으면 OK.

이상하면 아래 표를 따라가면 거의 다 잡힙니다.

| 증상 | 의심 원인 | 처치 |
| --- | --- | --- |
| 401 / 403 | API 키 오류, 잔액/권한 부족 | 콘솔에서 키와 잔액 재확인 |
| 404 | base URL 또는 모델명 불일치 | 최신 공식 문서로 값 갱신 |
| 서브에이전트만 다른 모델 호출 | `ANTHROPIC_DEFAULT_*_MODEL`, `CLAUDE_CODE_SUBAGENT_MODEL` 누락 | 모든 default 변수 동일 값으로 맞춤 |
| MCP/도구 검색 이상 | 비공식 엔드포인트 호환성 | `ENABLE_TOOL_SEARCH=false`로 시작 |
| 비용이 계속 증가 | 자동 재시도 루프 | 일일 예산/잔액 알림 먼저 활성화 |

## [6] 내가 지키는 운영 규칙

이 설정은 편하지만, **프로젝트 default로 박아두지는 않습니다.** 작업 성격에 따라 켜고 끕니다.

- 아이디어 정리, 초안 작성, 단순 리팩터링 → 대체 프로바이더 OK
- 보안 / 권한 / 결제 / 데이터 삭제 로직 → 기본 검증 환경 우선
- 장시간 자동화 → 예산 한도 먼저 설정하고 실행
- 결과가 애매하면 → 같은 작업을 Anthropic 기본 모델로 한 번 더 검증

## 총 정리

| 단계 | 해야 할 일 |
| --- | --- |
| 1. 발급 | DeepSeek API 키 + 일일 예산 설정 |
| 2. 세팅 | `ANTHROPIC_BASE_URL` / `_AUTH_TOKEN` / 모델 환경변수 동기화 |
| 3. 호출 | `claude` 실행 후 `/status`로 연결 확인 |
| 4. 굳히기 | PowerShell/Shell 프로필에 별도 함수로 등록 |
| 5. 운영 | 보안·결제·마이그레이션은 기본 환경, 나머지 실험 작업만 대체 환경 |

이 다섯 단계만 지키면 비용은 꽤 줄고, 중요한 작업의 품질도 안 잃습니다.
